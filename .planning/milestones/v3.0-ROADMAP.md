## Key Decisions

- Phase 09: CR-02 resolved via JOIN-free device_id propagation (per-row rather than session-level join)
- Phase 10: hrBluetoothState separate from bluetoothState (each CBCentralManager publishes its own state)
- Phase 11: Slot-based capture (activeHealthPacketCapture) — HR monitor and WHOOP share slot, independent in common case
- Phase 12: canSyncClock guard used (no Gen4 version filter — supportsClockCommands is sufficient)
- Phase 14: Separate display-layer extension (LocalizedStatusStrings.swift) — raw state values stay in English for logic
