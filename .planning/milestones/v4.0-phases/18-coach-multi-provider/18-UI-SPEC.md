## Spacing Scale

Standard 8-point scale mapped to SwiftUI numeric literals (project uses raw CGFloat, not tokens):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon-to-text gaps, status badge padding |
| sm | 8pt | HStack spacing inside rows, button vertical padding |
| md | 12–16pt | Card internal padding (12pt compact, 16pt standard) |
| lg | 24pt | Section breaks inside the settings sheet |
| xl | 32pt | Top padding on scroll content |
| 2xl | 48pt | Bottom safe-area inset, scroll content bottom |

Exceptions:
- Touch targets for icon-only toolbar buttons: minimum 44×44pt (SF Symbols in toolbar handled by system).
- Card internal padding 12pt is permitted for compact rows (existing pattern: `CoachSuggestionButton.padding(12)`, `CoachDataGapCard.padding(13)`). The existing 13pt value must NOT be replicated in new components introduced by this phase — use 12pt (compact) or 16pt (standard) instead.
- Sheet drag indicator adds natural vertical clearance; no extra top padding required.

Source: `CoachChatScreen.swift` (padding 16, 14, 92), `CoachView.swift` (padding 16, 18), `HRMonitorView.swift` (padding 24, 36, 48).

## Typography

All sizes use SwiftUI Dynamic Type text styles — these are the semantic mappings in use across Coach:

| Role | SwiftUI Style | Approx Size | Weight | Line Height |
|------|---------------|-------------|--------|-------------|
| Body | `.body` | 17pt | regular (400) | system (≈1.35) |
| Label / secondary | `.subheadline` | 15pt | regular or `.semibold` (600) | system |
| Caption | `.caption` / `.caption2` | 12–11pt | regular or `.semibold` | system |
| Heading / card title | `.headline` / `.title3` | 17–20pt | `.semibold` (600) | system (≈1.2) |

Rules:
- Two weights only: regular (400) and semibold (600). Never bold (700+) outside SF Symbol rendering.
- `.fontDesign(.rounded)` applied only to large numeric metric values (existing: `CoachMetricHighlightCard`).
- Monospaced font (`system(size: 11, weight: .regular, design: .monospaced)`) reserved for tool call payload blocks only — do not apply to settings fields.
- API key masked display uses `.font(.body)` with a `SecureField` or a masked replacement string — same weight as normal body text.
- For the new settings sheet, heading labels above grouped sections use `.caption.weight(.semibold)` in `.textCase(.uppercase)` (existing pattern: `CoachSuggestionStack` "Start Here" label, `CoachToolPayloadBlock` title label).

Source: `CoachMessageViews.swift`, `CoachView.swift`, `CoachChatScreen.swift` — all type usage extracted directly.

## Color

The app is theme-adaptive (dark/light) via `GooseTheme`. Semantic system colors are preferred over hardcoded hex.

| Role | Dark mode value | Light mode value | Usage |
|------|-----------------|------------------|-------|
| Dominant (60%) | `#0F1720` (rgb 0.06/0.09/0.11) | `.systemGroupedBackground` | Screen background via `.gooseScreenBackground()` |
| Secondary (30%) | `.secondarySystemGroupedBackground` | `.secondarySystemGroupedBackground` | Card surfaces, sheet background, List rows |
| Accent (10%) | `.blue` (system) | `.blue` (system) | Reserved elements listed below |
| Destructive | `.red` (system) | `.red` (system) | Sign-out action, remove-provider action only |

Accent (`.blue`) reserved for:
1. Active provider indicator badge in Coach tab header.
2. "Save" button for API key (`.buttonStyle(.borderedProminent)` which inherits tint).
3. OAuth sign-in buttons ("Sign in with ChatGPT", "Sign in with Google") — `.borderedProminent`.
4. Streaming and connection-active states in `CoachConnectionStrip`.
5. SF Symbol icon tint inside provider config rows when that provider is the active provider.

Provider-specific tints (10% of 10% — decorative only, not layout color):
- ChatGPT: `.green` — authenticated status badge.
- Claude: `.orange` — authenticated status badge.
- Gemini: `.blue` — authenticated status badge (shares accent, acceptable overlap).
- Custom: `.purple` — authenticated status badge.

Source: `GooseTheme.swift` (dominant/secondary), `CoachView.swift` `.coachCardSurface` extension (secondary surface), existing tint patterns throughout Coach.

## Component Inventory

New SwiftUI views required for this phase. Follow existing naming conventions (PascalCase, `Views` suffix for multi-view files, `+` extension file for extensions on existing types).

### CoachSettingsSheet

A `.sheet` presented from `CoachView` when the gear toolbar button is tapped.

```
CoachSettingsSheet
  NavigationStack
    List (grouped style)
      Section("Provider")
        CoachProviderPickerRow  ×4  (ChatGPT, Claude, Gemini, Custom)
      Section("Configuration")
        CoachProviderConfigView  (switches on active provider)
      Section("Model")
        CoachModelPresetPickerView
```

- Presented with `.presentationDetents([.large])` — settings sheet is full-height.
- `.presentationDragIndicator(.visible)`.
- Navigation title: "Coach Settings". `.navigationBarTitleDisplayMode(.inline)`.
- Toolbar: `ToolbarItem(placement: .topBarLeading)` → "Done" button dismisses sheet.

### CoachProviderPickerRow

A `Button` rendered as a List row showing:
- Leading: SF Symbol icon (see icon table below) + provider name (`.subheadline.weight(.semibold)`).
- Trailing: auth status badge (`.caption2.weight(.semibold)`) + `checkmark` if active.
- Tap selects this provider as active.

### CoachProviderConfigView

Switches on the active `CoachProvider.id`. Renders one of:

**ChatGPT config:**
- Auth status label (signed in or "Not signed in").
- "Sign in with ChatGPT" button (`.borderedProminent`, full width) — visible only when not authenticated. System image: `person.crop.circle.badge.checkmark`.
- "Sign Out" button (`.bordered`, role: `.destructive`) — visible only when authenticated. Triggers `.confirmationDialog("Sign Out?", isPresented: $showingSignOutConfirm)` with destructive "Sign Out" button and cancel.

**Claude config:**
- `SecureField("Anthropic API key", text: $apiKey)` — masked input. Trailing: lock icon (`.lock`).
- "Save API Key" button (`.borderedProminent`).
- Inline status label: "Key saved" / "No key saved" (`.caption`, `.secondary`).
- "Remove Key" button (`.bordered`, role: `.destructive`) — visible only when a key is saved.

**Gemini config:**
- Auth status label.
- "Sign in with Google" button (`.borderedProminent`) — presents the OAuth WKWebView.
- "Sign Out" button (`.bordered`, role: `.destructive`) when authenticated. Triggers `.confirmationDialog("Sign Out?", isPresented: $showingSignOutConfirm)` with destructive "Sign Out" button and cancel.

**Custom endpoint config:**
- `TextField("Base URL", text: $baseURL)` — plain style, `.keyboardType(.URL)`, `.autocorrectionDisabled()`.
- `SecureField("API Key", text: $apiKey)`.
- `TextField("Model ID", text: $modelID)` — e.g. `mistral-small`.
- "Save Endpoint" button (`.borderedProminent`).
- Inline URL validation hint: "Must start with https://" shown in `.caption` `.red` when URL format is invalid and field is non-empty.

### CoachModelPresetPickerView

A `Picker` rendered as `.pickerStyle(.inline)` inside a `List Section`. Shows only the presets for the currently active provider (derived from `CoachProvider.availablePresets`).

### Active Provider Indicator (CoachView header)

A small text label or badge appended below the "Coach" navigation title — or placed as a subtitle if iOS 26 navigation API supports it. If subtitle is not available, render as a `ToolbarItem(placement: .principal)` with:

```
VStack(spacing: 1) {
  Text("Coach").font(.headline.weight(.semibold))
  Text(activeProviderName).font(.caption2).foregroundStyle(.secondary)
}
```

This indicator is visible in `CoachView` when at least one provider is active. Hidden (empty string / not shown) when no provider is configured.

**Primary visual anchor:** The `CoachProviderPickerRow` active-provider `checkmark` is the primary visual anchor in the settings sheet — it communicates active selection state at a glance.

### Gear Icon Toolbar Button

`ToolbarItem(placement: .topBarTrailing)` in `CoachView`:
- `Image(systemName: "gearshape")` — standard SF Symbol, no label.
- Accessibility label: "Coach settings".
- Presented alongside existing `CoachProfileMenu` (which will become per-provider within the settings sheet in a later wave — for now, gear icon is separate).

## Interaction Contracts

### Provider Selection

- Tapping a `CoachProviderPickerRow` immediately sets that provider as active in `CoachProviderRegistry`.
- The `Configuration` section below updates to show that provider's config form without animation delay.
- No confirmation required for provider switch.

### API Key Entry (Claude / Custom)

- `SecureField` — text is masked with bullets at all times (iOS system behaviour).
- "Save API Key" is enabled only when the field is non-empty.
- On save: key written to Keychain, status label updates to "Key saved", field clears to empty (key not displayed again).
- "Remove Key" triggers an inline confirmation: `.confirmationDialog("Remove API key?", isPresented: $showingRemoveConfirm)` with a destructive button "Remove" and a cancel button. No full-screen alert.

### OAuth Sign-In (ChatGPT / Gemini)

- Tapping "Sign in with ChatGPT" / "Sign in with Google" triggers the existing OAuth flow (no new UI required beyond the button).
- During OAuth: button shows a `ProgressView` inline (`.controlSize(.small)`) and is disabled.
- On success: button replaced by status "Signed in" label + "Sign Out" button.
- On failure: inline error label in `.caption` `.red` below the button.

### Custom Endpoint Validation

- Base URL validated on "Save": must start with `https://`, must be a parseable URL.
- If validation fails: red `.caption` hint appears inline, "Save" button remains enabled (user can re-edit).
- On success: UserDefaults updated, inline "Saved" label appears for 2 seconds, then reverts to field.

### Sheet Dismissal

- "Done" button in sheet toolbar dismisses with `isPresented = false`.
- No unsaved-changes warning — all changes (provider selection, preset selection) are saved immediately on interaction.
- API key fields that have been edited but not saved: dismissed silently (field value discarded, no warning). This matches iOS convention for settings sheets.

## States and Empty/Error Handling

### Settings Sheet — No Provider Configured

When `CoachProviderRegistry.activeProvider == nil` (first launch after migration):
- Configuration section shows: `Text("Select a provider above to get started.")` `.foregroundStyle(.secondary)` `.font(.subheadline)`.
- Model section is hidden (no presets to show).

### Settings Sheet — Provider Active but Not Authenticated

- Config section shows the sign-in button (ChatGPT/Gemini) or API key field (Claude/Custom).
- Status badge in the picker row shows "Not signed in" in `.secondary` colour.

### Active Provider Indicator — No Provider

- The subtitle / principal indicator is hidden entirely. No empty string shown.

### Coach Chat — Provider Not Authenticated

- `CoachOverviewChatCard` (existing) adapts: status text shows "No provider configured" or "Provider not signed in".
- "Open" button is still visible but tapping it opens the chat screen which shows an inline notice (existing `errorMessage` display handles this).

## Accessibility

- All interactive rows have `.accessibilityLabel` set explicitly when the visual label is ambiguous (e.g. `SecureField` trailing lock icon).
- `SecureField` for API keys: iOS system provides "Secure text field" trait automatically — no override needed.
- Provider picker rows: mark selected row with `.accessibilityAddTraits(.isSelected)` when it is the active provider.
- "Save API Key" button: when disabled, set `.accessibilityHint("Enter an API key first")`.
- Touch targets: all List rows are system-managed (minimum 44pt height enforced by List). Custom buttons outside List must be at least 44×44pt.
