  <behavior>
    Types to define:
    - HrSample { ts: f64, bpm: u8 } — matches existing hr_samples JSON format
    - ExerciseProfile { resting_hr: Option&lt;f64&gt;, max_hr: Option&lt;f64&gt;, age: Option&lt;u8&gt;, sex: Option&lt;String&gt;, weight_kg: Option&lt;f64&gt;, height_cm: Option&lt;f64&gt;, daily_hr_p10: Option&lt;f64&gt; }
    - ExerciseSession { device_id: String, start_ts: f64, end_ts: f64, duration_s: f64, avg_hr: f64, peak_hr: f64, strain: f64, calories_kcal: f64, zone_time_pct: BTreeMap&lt;u8, f64&gt;, hrmax: f64, hrmax_source: String, rhr_source: String, avg_hrr_pct: f64 }

    Constants (pub, named, matching exercise.py exactly):
    - MIN_EXERCISE_MIN: f64 = 10.0
    - MERGE_GAP_S: f64 = 60.0
    - HR_MARGIN_BPM: f64 = 30.0
    - MOTION_THRESHOLD: f64 = 0.01
    - MOTION_SMOOTH_S: f64 = 3.0
    - ALIGN_TOLERANCE_S: f64 = 5.0
    - MIN_INTENSITY_Z2PLUS: f64 = 0.50

    Algorithm — detect_exercise_sessions(hr: &[HrSample], gravity: &[GravityRow], profile: &ExerciseProfile) -> Vec&lt;ExerciseSession&gt;:

    Step 1 — Resolve RHR: use profile.resting_hr if Some (rhr_source="profile_override");
      else use profile.daily_hr_p10 (rhr_source="daily_p10");
      else reject with empty Vec (cannot detect without RHR).
      Set rhr_source accordingly.

    Step 2 — Resolve HRmax: call resolve_effective_hrmax(profile.max_hr.unwrap_or(220.0 - profile.age.unwrap_or(30) as f64), profile.age.map(|a| a as f64), &[]).
      If hrmax_source == "fallback" AND profile.age is None: store hrmax_source as "fallback".
      Guard: if hrmax is NaN or <= rhr, return empty Vec.

    Step 3 — Smooth gravity: for each gravity row compute magnitude = sqrt(x^2 + y^2 + z^2) - 1.0 (remove 1g static).
      Apply rolling mean with window covering MOTION_SMOOTH_S seconds using nearest-timestamp window:
      for each sample i, collect all gravity rows within [ts_i - MOTION_SMOOTH_S/2, ts_i + MOTION_SMOOTH_S/2], compute mean magnitude.
      Use abs(mean_magnitude) as smoothed_mag for threshold gate.

    Step 4 — Align HR and smoothed gravity via nearest-neighbor within ALIGN_TOLERANCE_S:
      For each HR sample, find the gravity row with minimum |ts_gravity - ts_hr| <= ALIGN_TOLERANCE_S.
      Discard HR samples without a matching gravity row.
      Result: Vec of aligned pairs (ts, bpm, smoothed_gravity_mag).

    Step 5 — Apply dual-gate: keep aligned pair if bpm as f64 > rhr + HR_MARGIN_BPM AND smoothed_gravity_mag > MOTION_THRESHOLD.
      Active segments: runs of consecutive active pairs. Use timestamps to determine "consecutive" — gap between consecutive active samples <= MERGE_GAP_S counts as same segment.

    Step 6 — Merge adjacent segments with gap < MERGE_GAP_S s: iterate through active-pair runs;
      if gap between end of segment i and start of segment i+1 is < MERGE_GAP_S, merge.
      Continue until no more merges possible.

    Step 7 — Filter by minimum duration: drop merged sessions where (end_ts - start_ts) < MIN_EXERCISE_MIN * 60.0.

    Step 8 — Intensity gate: for each candidate session, compute Karvonen %HRR for each sample:
      hrr_pct = clamp((bpm - rhr) / (hrmax - rhr) * 100.0, 0.0, 100.0).
      Edwards 5-zone boundaries at 50%, 60%, 70%, 80%, 90% HRR:
        zone 1: hrr_pct < 50, zone 2: 50 <= hrr_pct < 60, zone 3: 60 <= hrr_pct < 70,
        zone 4: 70 <= hrr_pct < 80, zone 5: hrr_pct >= 80.
      fraction_z2plus = count(zone >= 2) / total_samples.
      Discard session if fraction_z2plus < MIN_INTENSITY_Z2PLUS.
      Guard: if hrmax is unknown (hrmax_source == "fallback" AND profile.age is None), skip intensity gate (keep session).

    Step 9 — Compute per-session ExerciseSession:
      duration_s = end_ts - start_ts.
      avg_hr = mean of bpm values in session.
      peak_hr = max bpm in session.
      avg_hrr_pct = mean of per-sample hrr_pct values.
      zone_time_pct: BTreeMap&lt;u8, f64&gt; with keys 1..=5, values = 100 * count_in_zone / total; always sums to 100 (use remaining fraction for zone 5 if floating point drift).
      strain: call goose_strain_v1(&StrainInput { duration_minutes, resting_hr_bpm: rhr, average_hr_bpm: avg_hr, max_hr_bpm: hrmax, hr_zone_minutes (from zone seconds/60), profile_sex: profile.sex.clone(), profile_age: profile.age.map(|a| a as f64), .. }). Extract score_0_to_21 from output.output; if None, use 0.0.
      calories_kcal: use energy_rollup helpers from crate::energy_rollup — call active EE (Keytel, samples where hrr_pct >= 30) and resting EE (H-B, samples where hrr_pct < 30). If energy_rollup not accessible, implement direct Keytel formula inline using exact Ghidra-confirmed coefficients (male: -55.0969 + 0.6309*hr + 0.1988*weight_kg + 0.2017*age; female: -20.4022 + 0.4472*hr - 0.1263*weight_kg + 0.0740*age; per minute, kJ/min / (60 * 4.184) = kcal/s).
      Clamp calories_kcal >= 0.

    Test cases that MUST pass (RED first, then GREEN):
    - test_alignment_within_tolerance: HR at ts=100, gravity at ts=103 → matched (3 s < 5 s). HR at ts=100, gravity at ts=106 → not matched (6 s > 5 s).
    - test_merge_gap_bridging: Two active segments with 45 s gap → merged. Two segments with 65 s gap → kept separate.
    - test_minimum_duration_filter: Session of 8 min rejected. Session of 11 min kept.
    - test_intensity_gate_discard: Session with 30% Z2+ time discarded (< 50%). Session with 55% Z2+ time kept.
    - test_zone_time_pct_sums_to_100: For any session, sum of zone_time_pct values == 100.0 (within 0.01 tolerance).
    - test_calories_positive: 30-min session at 70% avg HRR with weight_kg=75, age=30, sex="male" → calories_kcal > 0.
    - test_rhr_fallback_daily_p10: profile.resting_hr=None, daily_hr_p10=Some(50.0) → rhr_source="daily_p10" in output session.
    - test_no_rhr_no_sessions: profile.resting_hr=None, daily_hr_p10=None → returns empty Vec.
