<objective>
Implement ALG-SLP-01: compute HR-derived sleep quality metrics (heart_rate_dip_pct,
waso_minutes, sol_minutes, disturbance_count, rem_latency_minutes) using the
HR-threshold method specified in REQUIREMENTS.md, expose them on SleepScoreOutput,
wire them into the live scoring path, and surface them in the Sleep V2 dashboard.

Purpose: The existing sleep scorer derives WASO/SOL/HR-dip from a motion/stage-segment
heuristic (metric_features.rs::sleep_window_feature). ALG-SLP-01 requires the
physiologically-grounded HR-threshold method (HR vs resting_hr x 1.05 for wake
detection; rolling 5-min minimum for HR dip) as dedicated, unit-testable helpers
in metrics.rs, plus distinct output fields the dashboard can display.

Output: New pure helpers in metrics.rs, extended SleepScoreOutput, updated
sleep_window_feature wiring, and Sleep V2 detail UI rows.
</objective>

<task type="auto" tdd="true">
  <name>Task 1: HR-threshold sleep metric helpers in metrics.rs</name>
  <files>Rust/core/src/metrics.rs</files>
  <behavior>
    Add three pure helper functions (no DB, no I/O) with unit tests written first:
    - heart_rate_dip_pct(sleep_hr_series: &[(f64, f64)], baseline_awake_hr: f64) -> Option&lt;f64&gt;:
      computes the rolling 5-minute minimum HR over the in-bed window, returns
      (baseline_awake_hr - rolling_5min_min) / baseline_awake_hr * 100.0; None when
      baseline <= 0 or series empty; clamps negative dip (min above baseline) to 0.0.
      Test: known series with a clear nadir produces the hand-computed dip %; flat
      series at baseline produces 0.0; empty series produces None.
    - waso_from_hr(hr_series: &[(f64, f64)], resting_hr: f64, onset_ts: f64) -> f64:
      sums minutes where HR > resting_hr * 1.05, counting only samples at/after
      onset_ts; threshold uses the 1.05 factor per 24-CONTEXT specifics. Test: WASO = 0
      when no post-onset sample exceeds threshold; a 2-sample wake run yields the
      expected minutes; pre-onset wake samples are excluded.
    - sol_from_hr(hr_series: &[(f64, f64)], resting_hr: f64, sustained_minutes: f64) -> Option&lt;f64&gt;:
      returns minutes from window start to the first sustained low-HR period of
      >= sustained_minutes (default 3.0) where all HR samples <= resting_hr * 1.05.
      None when no sustained low-HR period exists. Test: a series that drops below
      threshold for >= 3 min returns the latency to that point; a series never sustaining
      low HR returns None.
    Also add a disturbance count helper hr_disturbance_count(hr_series, resting_hr, onset_ts) -> u32
    counting distinct post-onset runs crossing above resting_hr * 1.05 (>= 1 transition each).
    Test: contiguous wake run counts once; two separated runs count twice.
  </behavior>
  <action>
    Implement heart_rate_dip_pct, waso_from_hr, sol_from_hr, and hr_disturbance_count
    as pub functions in metrics.rs alongside the other metric helpers. Series are
    (timestamp_minutes, hr_bpm) tuples. Use the threshold factor 1.05 x resting_hr for
    wake detection and the rolling 5-min minimum for the dip nadir, per 24-CONTEXT
    <specifics>. Per ALG-SLP-01: SOL = first sustained low-HR period >= 3 consecutive
    minutes; WASO = wake epochs after SOL only. Keep all four functions side-effect-free
    so they are unit-testable in isolation. Write the #[cfg(test)] tests in metrics.rs
    BEFORE the implementations (RED), then implement to green. Do NOT change goose_sleep_v0
    scoring weights in this task.
  </action>
  <verify>

  </verify>
  <done>The four helpers exist in metrics.rs with passing unit tests covering nadir, WASO=0, SOL sustained-window, and disturbance run counting.</done>
