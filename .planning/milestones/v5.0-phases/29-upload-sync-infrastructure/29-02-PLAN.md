  <action>
    Add three store methods to GooseStore impl in store.rs:

    1. mark_synced_rows(stream: &str, row_ids: &[i64]) -> GooseResult<usize>

    Stream table allowlist (static slice): hr_samples, rr_intervals, events, battery, spo2_samples, skin_temp_samples, resp_samples, gravity. Reject any stream name not in the allowlist with GooseError::message("unknown stream: {stream}"). This prevents SQL injection via table name interpolation. If row_ids is empty, return Ok(0) immediately.

    Execute: UPDATE {stream} SET synced=1 WHERE rowid IN ({placeholders}) — build the parameterised placeholders as ?1,?2,... using the row_ids slice. Use rusqlite params_from_iter for the parameter binding. Return the count of affected rows.

    2. rows_pending_upload(stream: &str, limit: i64) -> GooseResult<Vec<serde_json::Value>>

    Same allowlist check as mark_synced_rows. Execute: SELECT rowid, * FROM {stream} WHERE synced=0 ORDER BY ts LIMIT ?1. Map each row to a serde_json::Value object. The rowid must be included as "rowid" key so callers can pass it back to mark_synced_rows. Return the vec.

    Note: Because each stream table has different columns, use dynamic column mapping via row.column_names() and row.get_ref(i) to build the JSON object — this avoids maintaining per-table struct variants in this method. Alternatively, use a generic helper that reads all columns via rusqlite's Row API. Either approach is acceptable; prefer the simpler one.

    3. backfill_streams_from_decoded_frames(device_id: &str, start_ts: f64, end_ts: f64) -> GooseResult<BackfillReport>

    BackfillReport struct: { hr_inserted: usize, rr_inserted: usize, events_inserted: usize, battery_inserted: usize }

    Logic: call self.decoded_frames_between(&chrono_from_unix_f64(start_ts), &chrono_from_unix_f64(end_ts)) to get frames in the window. For each frame where header_crc_valid && payload_crc_valid, deserialise parsed_payload_json as Option<ParsedPayload>. Extract HR bpm and RR intervals using the same match arms as upload_get_recent_decoded_streams_bridge (NormalHistory, RawMotionK10, V24History). Insert into hr_samples/rr_intervals using INSERT OR IGNORE (idempotent). The synced column defaults to 0, so backfilled rows are automatically pending upload — they are NOT stranded by any highwater cursor because rows_pending_upload uses WHERE synced=0 (not ts > highwater).

    For events and battery: the existing decoded_frames do not contain event or battery packets in this codebase phase. Insert 0 rows for both; return hr_inserted and rr_inserted from actual frame extraction.

    Wrap all inserts in a single immediate_transaction for atomicity.

    Also add the prune guard: in compact_raw_evidence_payloads_to_limit, the existing method only touches raw_evidence (NULLing payload_hex). This method does NOT delete from stream tables directly — stream table pruning is separate. Document this with a comment: "Stream table pruning (DELETE FROM hr_samples WHERE synced=1 AND ...) is intentionally NOT performed here — the invariant is enforced at the call site by the upload pipeline which checks synced=1 before any stream table DELETE." Add a separate store method: prune_synced_stream_rows(stream: &str, older_than_ts: f64) -> GooseResult<usize> that executes DELETE FROM {stream} WHERE synced=1 AND ts < ?1 using the same allowlist guard. This method can be called safely by the upload pipeline knowing it will never touch unsynced rows.

    Add 7 unit tests in a new #[cfg(test)] mod sync_methods_tests. The backfill test (test_backfill_stranded_guard) requires inserting a minimal decoded frame row directly via SQL to simulate a real frame — use the store's internal conn to INSERT a row into decoded_frames with a synthetic parsed_payload_json containing a DataPacket with NormalHistory body. Use the simplest possible JSON that round-trips through serde.
