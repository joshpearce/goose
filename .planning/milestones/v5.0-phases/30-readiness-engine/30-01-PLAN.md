<feature>
  <name>goose_readiness_v1 — ACWR + Foster monotony → 5-class readiness</name>
  <files>Rust/core/src/metrics.rs</files>
  <behavior>
    Input: ReadinessInput { daily_strain: Vec<(f64, f64)> } — (timestamp_secs_f64, strain_0_to_21_f64) pairs, newest-last ordering expected; caller provides trailing 28 days minimum.

    Insufficient data gate (RDY-01):
    - fewer than 28 entries → ReadinessOutput with level=Unknown, acwr=None, acwr_zone="unknown", monotony=None, monotony_high=false, insufficient_data=true

    ACWR computation (RDY-01):
    - acute_load  = mean of strain values in last 7 entries (entries[n-7..n])
    - chronic_load = mean of strain values in all 28 entries (entries[n-28..n])
    - acwr = (acute_load / chronic_load).clamp(0.0, 3.0); None if chronic_load == 0.0
    - acwr_zone string:
        < 0.8  → "under_training"
        0.8..1.3 (inclusive both ends) → "optimal"
        1.3..1.5 (exclusive 1.3, inclusive 1.5) → "caution"
        > 1.5 → "danger"
      Note: 0.8 boundary → "optimal" (not under_training); 1.3 boundary → "optimal" (not caution); 1.5 boundary → "danger" (not caution)

    Foster monotony (RDY-02):
    - window = last 7 strains (entries[n-7..n] values)
    - mean_7 = sum / 7
    - std_7  = population std dev of those 7 values (sqrt(mean of squared deviations from mean_7))
    - if std_7 == 0.0 OR fewer than 3 entries available → monotony = None, monotony_high = false
    - else → monotony = Some(mean_7 / std_7); monotony_high = monotony >= 2.0
    - Boundary: monotony exactly 2.0 → monotony_high = true

    Level synthesis (RDY-03):
    Evaluate in this priority order:
    1. Unknown: acwr is None (insufficient data already handled above by gate, but also chronic_load=0)
    2. Rundown: acwr >= 1.5  (danger zone triggers rundown regardless of monotony)
    3. Strained: monotony_high == true  (any high monotony signal = strained)
    4. Strained: acwr < 0.8  (under-training is a bad signal)
    5. Primed:   acwr >= 0.8 AND acwr <= 1.3 AND monotony_high == false
    6. Balanced: all other cases (acwr in caution range 1.3-1.5 exclusive, not monotony_high)

    Note: "strained when any single bad signal" — under-training (acwr<0.8) counts as bad even absent monotony.

    Boundary test cases that MUST pass:
    - 28 entries all strain=10.0 → acwr=1.0, zone="optimal", monotony=Some(0.0 or None?→std=0→None), level=Balanced (monotony None so no strained; acwr optimal)
    - Wait: all equal → std=0 → monotony=None → monotony_high=false; acwr=1.0 → Primed
    - 28 entries where last 7 avg=15.0, full 28 avg=10.0 → acwr=1.5, zone="danger", level=Rundown
    - acwr exactly 0.8 → zone="optimal"
    - acwr exactly 1.3 → zone="optimal"
    - acwr exactly 1.5 → zone="danger", level=Rundown
    - monotony exactly 2.0 (mean/std = 2.0) → monotony_high=true, level=Strained
    - 27 entries → insufficient_data=true, level=Unknown
    - 3 entries → insufficient_data=true (fewer than 28)
    - acwr=0.5 (under-training) → level=Strained

    Output struct: ReadinessOutput
    - algorithm_id: GOOSE_READINESS_V1_ID
    - algorithm_version: GOOSE_READINESS_V1_VERSION
    - acwr: Option<f64>
    - acwr_zone: String  ("under_training" | "optimal" | "caution" | "danger" | "unknown")
    - monotony: Option<f64>
    - monotony_high: bool
    - level: ReadinessLevel
    - insufficient_data: bool

    ReadinessLevel enum — serde rename_all = "snake_case":
    - Rundown, Strained, Balanced, Primed, Unknown
  </behavior>
