# Milestone v7.0: Sync Correctness, Async & Sleep Sync

**Status:** ✅ SHIPPED 2026-06-10
**Phases:** 46-50 (Phase 51 deferred — hardware gate)
**Total Plans:** 18 plans (Phases 46-50)

## Overview

Este milestone entregou três capacidades críticas: (1) upload/export round-trip completo de raw BLE frames para o servidor self-hosted, (2) resolução do mismatch de identificadores de dispositivo UUID vs device_model, (3) migração completa do HealthDataStore de síncrono para async/await eliminando freeze risk na main thread, e (4) sincronização matinal automática de dados de sono da pulseira WHOOP via Cole-Kripke pipeline.

**Known gaps at close:** Phase 51 (Validation Gates) deferred — requer dispositivo WHOOP físico + ≥5 noites de captura overnight. VAL-HRV-01 e VAL-SLP-01 permanecem como human gates.

## Milestone Summary

**Key Decisions:**

- device_uuid uses Option<&'a str> on RawEvidenceInput (not Option<String>); index references captured_at; PRAGMA user_version not bumped; backward compatible
- Phase 49 Plan 01: requestAsync/requestValueAsync added as additive wrappers (Task.detached) so sync FFI never runs on @MainActor
- CR-01 fix: BLE polling uses "synced" (not "complete") to detect historical sync completion
- V24History gravity extraction added inline to upload_get_recent_decoded_streams_bridge (K10 pattern analog)
- syncBandSleepHistory uses own GooseRustBridge() instance to avoid data races with GooseAppModel.rust shared instance
- Morning sync trigger: connectionState "ready" + hour≥4 + UserDefaults date guard (idempotent)
- "goose_ble" added to ALLOWED_EXTERNAL_SLEEP_PLATFORMS; sleep_id format: "band_ble.{deviceId}.{yyyy-MM-dd}"
- Algorithm defaults promoted: sleep v1, strain v1, recovery v1; readiness v1 added to defaults

**Issues Resolved:**

- Upload sync race condition (blind-marking) eliminated
- HealthDataStore @MainActor bridge freeze risk eliminated
- Missing server endpoints (ROUTE-01, ROUTE-02) implemented
- device_uuid mismatch between iOS and server resolved

**Issues Deferred:**

- Phase 51 (hardware gate) — VAL-HRV-01, VAL-SLP-01 require physical WHOOP + overnight data
- MoreDataStore sync bridge callers (18+) — not in Phase 49 scope
- Swift XCTest SYNCR-01 upload assertions skip (decoded_frames seeding needed)

**Technical Debt:**

- MoreDataStore.swift retains 18+ sync bridge.request callers (not @MainActor risk currently)
- GooseAppModel+Upload 2 sync bridge callers retained (background queue context)
- Swift XCTest SYNCR-01 HTTP assertions skip — Rust contract test covers underlying behavior
