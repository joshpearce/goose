<objective>
Wire the Wave-1 `GooseBLEBondingManager` into the live BLE flow (BLE-BOND-01) without breaking the string-based `connectionState` API surface. `GooseBLEClient` gains the manager as a `let` property; its `onBondingStateChange` callback drives the existing `updateConnectionState()` bridge so all 33 `connectionState ==` comparison sites keep working. The four bonding-path transitions (connecting/discovering/connected/ready) route through `bondingManager.transition(to:)`. Bond loss (CBError 14 / CBATTError 15) is detected at disconnect and re-enters the bonding flow automatically. `GooseAppModel` exposes a computed `bondingState` for observability.

Purpose: Make the formal bonding lifecycle the single source of truth for the connection portion of `connectionState`, satisfying Success Criteria 1, 2, and 4, while Phase 65 handles the full string→enum migration later.
Output: bonding manager integrated into GooseBLEClient + delegate + commands; observable from GooseAppModel; bond loss auto-recovery in place.
</objective>
