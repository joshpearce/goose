<objective>
Create the `GooseUploadWatermark` persistence store: a small Swift type that reads, writes, and clears two independent upload high-water-mark timestamps in `UserDefaults` under the `goose.swift.upload.*` namespace.

Purpose: Provide the crash-safe per-type watermark primitive that Wave 2 wires into the upload pipeline. Currently `sinceTimestamp` is a session-only value (`lastUploadAt ?? now - 24h`) that resets on restart, causing large re-uploads after a crash. This store persists the last confirmed upload point so the next launch resumes from it. Implements UPLOAD-WM-01 (persistence + reset-path primitives).

Output: `GooseSwift/GooseUploadWatermark.swift` — a self-contained, dependency-free type with no behavioural side effects on the rest of the app. Wave 2 (62-02) consumes it.
</objective>

Create a new file `GooseSwift/GooseUploadWatermark.swift` containing a `WatermarkType`
enum and a `GooseUploadWatermark` type that persists two independent `Date` watermarks
to `UserDefaults.standard`.

Follow the exact conventions in `GooseBLEBondingManager.swift`: `static let` string
constants for keys, dot-namespaced reverse-DNS strings, 2-space indentation, no `///`
doc comments (use `//` for non-obvious logic only).

Required shape:
- `enum WatermarkType { case rawFrames; case decodedStreams }` — two independent types
  (per RESEARCH Pitfall 3, raw and decoded uploads fail independently and MUST NOT share
  a watermark).
- Two key constants on the store:
  `goose.swift.upload.rawFramesWatermark` and
  `goose.swift.upload.decodedStreamsWatermark` (per CONTEXT preferred namespace
  `goose.swift.upload.*`).
- `static func watermark(for type: WatermarkType) -> Date?` — reads via
  `UserDefaults.standard.object(forKey:) as? Date` (NOT `double(forKey:)` — see RESEARCH
  Pitfall 5). Returns nil when never written.
- `static func update(_ type: WatermarkType, to date: Date)` — writes via
  `UserDefaults.standard.set(date, forKey:)`. Add a `//` comment stating this must be
  called ONLY inside an upload-success (2xx) branch, never on failure.
- `static func clearAllWatermarks()` — removes both keys via `removeObject(forKey:)`
  (the logout / device-swap reset path required by Success Criterion 4).
- A `private static func key(for type: WatermarkType) -> String` mapping each case to its
  constant (single source of truth for the switch).

Use an `enum` with static members (no instances needed — the store is stateless, state
lives in UserDefaults), matching the stateless pattern noted in RESEARCH A1. Import only
`Foundation`. Do NOT introduce any external dependency (CLAUDE.md constraint: URLSession/
Foundation only). Do NOT add OSLog or any logging in this file — pure persistence.
