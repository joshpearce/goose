<objective>
Promote the HR monitor BLE state (`discoveredHRDevices`, `hrConnectionState`) to `@Published` properties on `GooseBLEClient` so SwiftUI can observe them reactively and thread-safely, fix the missing `"connecting"` intermediate state, add a complete `disconnectHRMonitor()` action, and add the missing `didFailToConnect` delegate so a failed connection does not leave the UI stuck. Establish the test scaffold this phase's Nyquist map depends on.

Purpose: This is the reactive-state foundation that `HRMonitorView` (Plan 02) renders. Without these `@Published` mirrors the view cannot observe scan results or connection state, and the data race flagged in STATE.md (`discoveredHRDevices` BT queue vs. main thread, HIGH severity) is unresolved. Implements WEAR-04 (observable scan list) and WEAR-05 (observable connection lifecycle) at the model tier.

Output: Two edited Swift source files plus one new test file (`HRMonitorStateTests.swift`), the Wave 0 dependency for the whole phase.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BLE peripheral -> app | Advertised device name and RSSI from untrusted nearby BLE devices cross into app state |
| CoreBluetooth queue -> main thread | BLE callbacks run on the CoreBluetooth queue; UI state lives on the main thread |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | `centralManager(_:didDiscover:)` device name | mitigate | Existing `String(rawName.prefix(64))` + whitespace trim already bounds and sanitises advertised names before they enter `discoveredHRDevices`; this plan preserves that sanitisation unchanged (ASVS V5) |
| T-10-02 | Spoofing / DoS | `discoveredHRDevices` array shared across BLE queue and main thread | mitigate | Promote to `@Published` and mutate `owner?.discoveredHRDevices` only inside `DispatchQueue.main.async` (resolves STATE.md HIGH-severity data race) |
| T-10-03 | DoS (stuck UI) | connection failure with no `didFailToConnect` handler | mitigate | Add `didFailToConnect` resetting `hrConnectionState` to `"disconnected"` (ASVS V7 error handling) |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — Apple system frameworks only; RESEARCH Package Legitimacy Audit: "Not applicable" |
