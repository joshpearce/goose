# Phase 12 Plan 01 — Auto-trigger RTC Clock Sync on WHOOP Ready

## Objective

Add a 4-line inline call in `GooseAppModel+Lifecycle.swift` so that when the WHOOP device transitions to "ready" state, the app automatically sends GET_CLOCK (and auto-SET_CLOCK when drift > 5s). The full pipeline already exists in `GooseBLEClient` — only the trigger is missing.

## Tasks

### Task 1 — Add clock sync trigger in handleBLEConnectionStateChange

**File:** `GooseSwift/GooseAppModel+Lifecycle.swift`

After `scheduleAutoStartRespiratoryPacketWatchIfNeeded()` in the `guard state == "ready"` block, add:

```swift
if ble.canSyncClock {
  ble.writeClockCommand(.get, syncIfNeeded: true)
  ble.record(source: "ble.clock", title: "clock.auto_sync.triggered", body: "state=ready")
}
```

### Task 2 — Verify Rust tests still pass

Run `cargo test -p goose-core` to confirm no regressions.

- `grep -c "clock.auto_sync.triggered\|writeClockCommand(.get" GooseSwift/GooseAppModel+Lifecycle.swift` → ≥1
- `grep -c "canSyncClock" GooseSwift/GooseAppModel+Lifecycle.swift` → 1
- Existing `writeClockCommand` call sites in DeviceView unchanged
- cargo test passes
