# Plan 04-01: Expose Upload Status @Published Properties on GooseAppModel+Upload

## Objective

Add three `@Published` status properties ‚Äî `serverReachable: Bool?`, `lastUploadAt: Date?`, and `pendingBatchCount: Int` ‚Äî to `GooseAppModel+Upload.swift` (created in Phase 3), and implement the one-shot health check (`GET /healthz`) that fires at app launch when upload is enabled and the server URL is configured.

## Context

Phase 3 creates `GooseAppModel+Upload.swift` with `GooseUploadService` integration. Phase 4 extends it to surface upload status for the UI. The health check runs once on `scenePhase == .active` (first launch) if `uploadEnabled == true` and server URL is set. The result is published on the `@MainActor` via the `@Published` property `serverReachable`.

<threat_model>
## Threat Model (ASVS L1)

| Threat | Mitigation |
|--------|-----------|
| Health check response body parsed as user data | Response is discarded except for HTTP status code 200 check |
| Server URL injected with malformed bytes | URLSession rejects malformed URLs; health check bails early if `URL(string:)` returns nil |
| Health check called repeatedly (DoS self-inflicted) | Boolean `didRunHealthCheck` guard ensures one-shot per app session |
</threat_model>

<action>
Add to `GooseAppModel+Upload.swift` (inside the extension on `GooseAppModel`):

Three `@Published` status properties on the `@MainActor` class (following `@Published private(set) var` pattern from GooseAppModel):
- `@Published private(set) var serverReachable: Bool? = nil` ‚Äî nil until health check completes
- `@Published private(set) var lastUploadAt: Date? = nil` ‚Äî set by GooseUploadService after successful upload
- `@Published private(set) var pendingBatchCount: Int = 0` ‚Äî set by GooseUploadService during retry

Add a private boolean guard `private var didRunHealthCheck = false` to ensure one-shot semantics (D-03: health check runs once per app session; no periodic re-checks).

Add a method `func triggerHealthCheckIfNeeded()` that:
1. Guards `!didRunHealthCheck`, `uploadEnabled == true`, and server URL is non-empty (read from UserDefaults key `"goose.remote.serverURL"`)
2. Sets `didRunHealthCheck = true`
3. Dispatches to a background queue (`DispatchQueue.global(qos: .utility)`)
4. Constructs `URL(string: serverURLString + "/healthz")` ‚Äî bail if nil, set `serverReachable = false` on main
5. Creates `URLRequest(url:)` with `timeoutInterval: 5`
6. Calls `URLSession.shared.dataTask(with:)`, checks `(response as? HTTPURLResponse)?.statusCode == 200`
7. Dispatches result back to main actor: `Task { @MainActor in self.serverReachable = isReachable }`
8. Logs result via `ble.record(level: .debug, source: "upload.health", title: "healthz", body: "reachable=\(isReachable)")`

Hook `triggerHealthCheckIfNeeded()` into `handleAppLifecycleChange("active")` in `GooseAppModel+Lifecycle.swift` ‚Äî call it only once (the `didRunHealthCheck` guard handles idempotency).
</action>
