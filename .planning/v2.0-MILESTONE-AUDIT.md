## Critical Gap: parse_hr_measurement Not Integrated

**Blocker:** `Rust/core/src/heart_rate_gatt_protocol.rs` exports `parse_hr_measurement` which is tested in isolation (10 tests, all pass). However, it is **never called** from `upload.get_recent_decoded_streams` in `bridge.rs`. When an HR monitor triggers a manual upload:

1. BLE bytes arrive → `notificationIngestResult` HR_MONITOR bypass → stored raw in SQLite ✓
2. `triggerManualUpload` calls `upload.get_recent_decoded_streams` → searches for `ParsedPayload::DataPacket` variants (WHOOP protocol) → finds none for HrMonitor frames
3. Upload payload has `device_class: "HR_MONITOR"` but `hr: []`, `rr: []`, all streams empty

**Fix required:** `upload.get_recent_decoded_streams` needs an HrMonitor-specific path that calls `parse_hr_measurement` on the raw stored GATT bytes and extracts hr_bpm + rr_intervals into the decoded stream arrays.
