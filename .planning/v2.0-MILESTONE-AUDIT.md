## Critical Gap: Phase 8.1 CR-02 Filter Regression (BLOCKER-1)

Phase 8.1 replaced the documented no-op CR-02 comment with a real filter that compares incompatible namespaces:
- `args.device_id` = `deviceID.uuidString` (CoreBluetooth UUID, e.g., "A1B2C3D4-E5F6-...")
- `evidence.device_model` = `ble.activeDeviceName` (BLE device name, e.g., "WHOOP 4B4A3")

Since `device_id` is always non-empty (UUID), every frame is discarded. This breaks **all uploads** — both WHOOP and HR monitor.

**Fix:** Revert the CR-02 filter to a no-op for the single-device case (add a comment explaining the namespace mismatch for v3.0 multi-device tracking).
