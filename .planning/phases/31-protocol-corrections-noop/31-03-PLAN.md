# Phase 31 Plan 03: PROTO-03 Graceful Resp Degradation in Sleep Staging

## Objective

Make the 4-class sleep classifier degrade gracefully when respiratory data is absent: when resp_samples has no rows for a session, suppress REM classification and return "light" for would-be REM epochs.

## Tasks

### Task 1 (auto): sleep_staging.rs — add resp_available param

- Add `resp_available: bool` parameter to `stage_sleep_four_class`
- Add `resp_available: bool` parameter to `classify_sleep_epoch`
- Gate REM classification on `resp_available` in `classify_sleep_epoch`
- Update all test call sites (pass `true` for existing tests, `false` for new test)
- Add `four_class_no_resp_suppresses_rem` test

### Task 2 (auto): store.rs — resp_samples_between method

- Add standalone `resp_samples_between` for use by the bridge to check resp presence

### Task 3 (auto): bridge.rs — wire resp_available in sleep staging bridge

- Add `resp_available: bool` field to `SleepStagingBridgeArgs` with `default_resp_available()=true`
- When `resp_available=true`, query `resp_samples_between` to check actual presence
- Pass result to `stage_sleep_four_class`
