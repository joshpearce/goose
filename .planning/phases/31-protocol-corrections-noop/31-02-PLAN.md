# Phase 31 Plan 02: PROTO-02 V24 gravity2 Triplet

## Objective

Add a second gravity triplet (gravity2_x/y/z) to the V24History protocol struct, parse it from bytes 49-57 of the V24 body, persist it to a new gravity2_samples table, and expose it via bridge methods.

## Tasks

### Task 1 (auto): protocol.rs — add gravity2 fields to V24History

- Add `gravity2_x: Option<f32>`, `gravity2_y: Option<f32>`, `gravity2_z: Option<f32>` to `DataPacketBodySummary::V24History`
- Parse from `data[49]`, `data[53]`, `data[57]` when `data.len() >= 60`
- Both short-payload and full-payload branches updated

### Task 2 (auto): store.rs — gravity2_samples table and methods

- Schema v18 → v19: create `gravity2_samples` table with `(device_id, ts, x, y, z, created_at, UNIQUE(device_id,ts))`
- Add to `known_tables()`
- Add `insert_gravity2_batch` and `gravity2_samples_between` methods (mirror of gravity equivalents)
- Update schema version test

### Task 3 (auto): bridge.rs — dispatch arms

- Add `store.insert_gravity2_batch` and `store.gravity2_samples_between` to KNOWN_METHODS
- Add dispatch match arms and handler functions

### Task 4 (auto): storage_check.rs — column registry

- Add `gravity2_samples` entry with columns `[device_id, ts, x, y, z, created_at]`
