<objective>
Add `DataPacketBodySummary::V26PpgWaveform` variant to protocol.rs and parse packet_k 26
(WHOOP 5.0 type-47 v26 PPG waveform — 88B fixed-layout packet) out of the Unknown catch-all.

Purpose: Every WHOOP 5.0 sync currently discards 24-sample PPG waveform bursts (26× per second)
silently. This plan makes those frames parseable so Phase 113 can persist them.

Output:
- New `V26PpgWaveform { ppg_channel: u8, unix_ts: u32, samples: Vec<i16> }` enum variant
- New `parse_v26_ppg_body` private function (hardware-verified byte layout)
- Match arm for packet_k 26 in `parse_data_packet_body_summary`
- Updated `data_packet_domain()` for packet_k 26
- Public test accessor `parse_v26_ppg_body_for_test` following the `parse_v24_body_for_test` pattern
- Synthetic 88B fixture tests in `protocol_tests.rs`
</objective>
