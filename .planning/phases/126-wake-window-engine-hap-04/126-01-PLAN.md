# Plan 126-01: Wake-Window Engine (HAP-04)

## Context

`GooseWakeWindowManager` is a stub actor with no methods. The full BLE alarm stack is already implemented:
- `BLETransport` protocol declares `setWhoopAlarm(at:alarmID:)` at line 153 and a convenience overload at line 202.
- `CoreBluetoothBLETransport+UserActions.swift` implements `setWhoopAlarm(at:alarmID:)` which calls `writeAlarmCommand(.set(...))`.
- `writeAlarmCommand` in `CoreBluetoothBLETransport+Commands.swift` encodes the SET_ALARM_TIME (0x42) frame and guards the connected state.
- `CoachRouteViews.swift:191` already calls `model.ble.setWhoopAlarm(at: alarmTime)` — no change needed.

This plan wires the actor stub to the existing stack. No new payload assembly. No protocol changes.

## Wave 1 — Implement GooseWakeWindowManager

### Task 1.1 — Replace stub with functional actor

**File:** `GooseSwift/GooseWakeWindowManager.swift`

Replace the entire file content. The actor:
- Holds `private weak var ble: (any BLETransport)?` — `weak` to avoid retain cycle; `any BLETransport` because the protocol is `AnyObject`.
- Init accepts `ble: any BLETransport`.
- `func armAlarm(target: Date)` calls `ble?.setWhoopAlarm(at: target)` which delegates through the existing convenience overload → `setWhoopAlarm(at:alarmID:1)` → `writeAlarmCommand(.set(..., pattern: .whoopDefault))`.
- No guard for connected state — `writeAlarmCommand` already guards `connectionState == "ready"`, `activePeripheral != nil`, `supportsAlarmCommands`, etc. The actor does not duplicate those checks.

**Implementation:**

```swift
import Foundation

// HAP-04: Wake-Window Engine
//
// Delegates alarm arming to BLETransport.setWhoopAlarm(at:), which assembles
// the SET_ALARM_TIME (0x42) frame and writes to CMD_TO_STRAP.
// Connected-state guard is enforced by writeAlarmCommand inside the transport layer.
actor GooseWakeWindowManager {
  private weak var ble: (any BLETransport)?

  init(ble: any BLETransport) {
    self.ble = ble
  }

  func armAlarm(target: Date) {
    ble?.setWhoopAlarm(at: target)
  }
}
```

**Done criteria:**
- File compiles with no errors.
- Actor has `init(ble:)` and `armAlarm(target:)`.
- No stub comment remains.
