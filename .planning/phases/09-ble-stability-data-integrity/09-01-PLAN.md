<objective>
Make the Rust FFI boundary panic-safe (FIX-04) and expose the already-implemented storage compaction algorithm as a bridge method (FIX-05, Rust side).

Purpose: A malformed bridge request that triggers a Rust panic currently aborts the entire iOS app process. Wrapping the FFI entry point in `catch_unwind` (after switching the release profile to `panic = "unwind"`) turns any panic into a structured JSON error. Separately, the 24 MB storage retention cap requires a bridge method that Swift can call; the compaction algorithm already exists in `store.rs` and only needs wiring.

Output: `panic = "unwind"` in the release profile, a `catch_unwind` wrap on `goose_bridge_handle_json`, a new `storage.compact_raw_evidence` bridge method, and Rust tests proving both behaviors.
</objective>
