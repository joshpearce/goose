<objective>
Add GET_FF_VALUE (cmd 0x80) auto-discovery to the BLE handshake path: extend DeviceCapabilities
with a featureFlags field, send the command after GET_HELLO on every reconnect with a 3-second
timeout fallback, parse the response into index→value pairs, and persist them to SQLite via the
existing capabilities.upsert_feature_flags bridge method.

Purpose: Closes FF-01 and the persistence/parsing half of FF-02 (#165). The Rust side
(device_feature_flags table, upsert/get bridge methods) is already complete from Phase 113;
this plan is Swift-only.
Output: DeviceCapabilities.featureFlags field, sendGetFeatureFlagValue() send + timeout,
handleFeatureFlagValue() parser + bridge write, and unit tests.
</objective>
