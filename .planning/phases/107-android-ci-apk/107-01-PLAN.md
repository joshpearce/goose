# Plan 107-01: Enable APK CI Step in android-core.yml

## Objective

Uncomment the two existing commented APK build blocks in `.github/workflows/android-core.yml`,
add Java 21 setup (required for AGP 9.2 jvmToolchain JDK discovery in CI), update the NDK
package from `27.2.12479018` to `28.2.13676358`, and add Android SDK platform + build-tools
packages. After this change, every `v*` tag push produces `app-release-unsigned.apk` attached
to the GitHub release.

## Context

- `.github/workflows/android-core.yml` already has the APK steps as comments (lines 81–91).
- The `android/` project builds locally: `./gradlew assembleDebug` BUILD SUCCESSFUL (commit 9511dfa).
- AGP 9.2.0, Kotlin 2.4.0, Gradle 9.4.1 — all current and building locally.
- `gradle.properties` has no JDK pin — CI must provide Java 21 via `actions/setup-java@v4`
  because AGP 9.2's jvmToolchain resolution requires JDK discovery via the action.
- NDK must be updated from `27.2.12479018` → `28.2.13676358` (AGP 9.2 default NDK).
