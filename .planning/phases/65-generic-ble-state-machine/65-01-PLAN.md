<objective>
Extract a minimal, reusable `StateMachine<State: Hashable, Event>` value type and migrate `GooseBLEBondingManager`'s bonding lifecycle onto it, matching the WHPStateMachine pattern. The bonding manager's public API (`bondingState`, `transition(to:)`, `onBondingStateChange`) stays byte-for-byte compatible so none of the 33 `connectionState == "..."` comparison sites or the 11 `bondingManager.transition(to:)` / callback call sites break.

Purpose: Replace ad-hoc untyped state tracking inside the BLE bonding layer with a typed, validated state machine that asserts illegal transitions in DEBUG and logs them via OSLog in RELEASE — without changing any observable behaviour. Implements SM-01.

Output: New `GooseStateMachine.swift` (generic struct), a `GooseBLEBondingEvent` enum + transition table in `GooseBLETypes.swift`, a refactored `GooseBLEBondingManager` whose `bondingState` is driven by an internal `StateMachine`, and the corresponding `project.pbxproj` wiring for the new file.
</objective>
