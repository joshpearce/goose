`func handleHRConnectionStateChange(_ state: String)`:
- If `state == "connected"`: call `startHRMonitorCapture(source: "auto.hr_monitor_connected")` (D-02 source string). Record `ble.record(source: "health.packet_capture", title: "hr_monitor.auto_start", body: state)`.
- Else if `state == "disconnected"`: call `stopHRMonitorCapture(reason: "hr_monitor_disconnected")`. Record `ble.record(source: "health.packet_capture", title: "hr_monitor.auto_stop", body: state)`.
- Ignore any other value (e.g. "connecting").

Do NOT add a WHOOP-capture guard — HR monitor capture runs in parallel with any active WHOOP capture (D-02). The `activeHealthPacketCapture == nil` guard inside `startHRMonitorCapture` (from Plan 01) is the only gate; if a WHOOP capture already holds the slot, the HR monitor auto-start is rejected by that guard rather than corrupting state. (Note: `activeHealthPacketCapture` is a single slot shared by both capture families — this is the existing design; parallel WHOOP+HR capture is possible only when no WHOOP capture currently owns the slot. Document this limitation in the SUMMARY; it satisfies success criterion 5 "both run independently" for the common case where they do not contend for the slot simultaneously.)

```
ble.onHRConnectionStateChange = { [weak self] state in

  }
}
```
