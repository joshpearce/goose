<objective>
Add the `.hrMonitor` capture mode and a dedicated start/stop method pair to `GooseAppModel`, enabling HR monitor BLE frames (standard GATT 2A37 BPM + RR) to be captured and persisted with NO WHOOP session requirement.

Purpose: Decouple HR monitor capture from the WHOOP `connectionState == "ready"` gate (D-01, D-03). Today, HR monitor frames flow through the notification pipeline but are silently discarded because `activeHealthPacketCapture == nil` — only a WHOOP-gated capture can set it. This plan adds the path that sets `activeHealthPacketCapture` from an HR-monitor-only trigger.

Output: A compiling `.hrMonitor` enum case (all 4 computed properties) and `startHRMonitorCapture(source:)` / `stopHRMonitorCapture(reason:)` methods. Existing WHOOP capture paths (`.walk`, `.temperature`, `.physiology`) are UNCHANGED.
</objective>
