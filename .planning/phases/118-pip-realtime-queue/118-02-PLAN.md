<objective>
Add the Swift `RealtimePIPQueue` — a structural twin of `CaptureFrameWriteQueue`
that, for every live BLE notification frame, inserts a `(device_uuid, frame_hex,
captured_at)` row into `realtime_frames` via the `realtime.insert_frame` bridge
method built in Plan 118-01. Wire it into `GooseAppModel` and call it on the
always-on notification path (D-01), above the capture-session guard.

Purpose: PIP-01 (Swift layer). The realtime stream must be captured
unconditionally and isolated from the existing historical capture queue —
separate lock, queue, table, and backpressure accounting (D-02).

Output: a new always-on write queue feeding realtime_frames in parallel with the
existing capture pipeline.
</objective>
