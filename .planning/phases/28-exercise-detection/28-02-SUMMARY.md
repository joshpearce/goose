# Phase 28 Plan 02: Schema Migration v17 (exercise_sessions) Summary

SQLite schema migration v17 adding the exercise_sessions table, ExerciseSessionRow struct, two store methods, and storage_check registration.

## What Was Built

- **CURRENT_SCHEMA_VERSION** bumped from 16 to 17 in `Rust/core/src/store.rs`
- **exercise_sessions table DDL** added to the `migrate()` SQL batch (after sig_quality_samples block):
  - Primary key: `session_id TEXT` with `DEFAULT (lower(hex(randomblob(16))))`
  - Unique constraint: `UNIQUE(device_id, start_ts)` — idempotent insert guard
  - Index: `idx_exercise_sessions_device_ts ON exercise_sessions(device_id, start_ts)`
  - All numeric metrics stored as `REAL NOT NULL` with sensible defaults
  - `zone_time_pct_json TEXT NOT NULL DEFAULT '{}'` for Edwards zone distribution
  - `hrmax_source` and `rhr_source` provenance fields with defaults
- **Migration records**: `INSERT OR IGNORE INTO goose_schema_migrations(version) VALUES (17)` + `PRAGMA user_version = 17`
- **ExerciseSessionRow struct** (12 fields: device_id, start_ts, end_ts, duration_s, avg_hr, peak_hr, strain, calories_kcal, zone_time_pct_json, hrmax_source, rhr_source, avg_hrr_pct)
- **insert_exercise_session** method: validates device_id, wraps in `immediate_transaction`, uses `INSERT OR IGNORE`, returns `bool` (true=inserted, false=duplicate)
- **exercise_sessions_between** method: validates device_id and ts ordering, returns `Vec<ExerciseSessionRow>` ordered by start_ts ascending
- **known_tables()** updated with "exercise_sessions" entry in store.rs
- **required_columns()** in storage_check.rs updated with all 14 exercise_sessions columns
