<objective>
Establish the localisation infrastructure for the GooseSwift app and translate the
top-level navigation surface (tab bar, More tab routes) to pt-PT.

This plan creates the `Localizable.xcstrings` String Catalog, registers `pt-PT` in the
Xcode project so the locale is compiled into the bundle, and fixes the three
`String`-returning computed properties (`GooseAppTab.title`, `MoreRoute.title`,
`MoreRoute.subtitle`) that bypass SwiftUI auto-localisation.

Purpose: Without the catalog file and the `pt-PT` knownRegion, none of the static
`Text("...")` strings in later waves will resolve — this plan is the foundation every
subsequent wave depends on (per D-02 and D-05 Wave 1).
Output: `Localizable.xcstrings`, updated `project.pbxproj`, localised tab + More-route titles.
</objective>

<task type="auto">
  <name>Task 2: Fix String-returning title/subtitle properties to use String(localized:)</name>
  <files>GooseSwift/AppShellView.swift, GooseSwift/MoreRouteModels.swift</files>
  <action>
These three computed properties return raw `String` literals, so they bypass SwiftUI catalog
auto-resolution (Pitfall 1). They must NOT change return type to `LocalizedStringKey` because
several call sites consume them as plain `String` — `CoachTips.swift` interpolates
`"\(route.title) Coach"` and calls `route.title.lowercased()`, `HealthScoreDateViews.swift`
maps `"\($0.route.title) ..."`, and `HomeDashboardView.swift` passes `route.title` as a `String`
`detail:` argument. Changing the type would break compilation. Use `String(localized:)`
(Pattern 4 Option B) so the return type stays `String` (D-03: do not change semantics).

1. `GooseSwift/AppShellView.swift` `GooseAppTab.title` (switch ~line 90): wrap each returned
   literal in `String(localized:)` — e.g. `case .home: String(localized: "Home")` for all four cases.
2. `GooseSwift/MoreRouteModels.swift` `MoreRoute.title` (switch ~line 22): wrap each of the 15
   returned literals in `String(localized:)`.
3. `GooseSwift/MoreRouteModels.swift` `MoreRoute.subtitle` (switch ~line 42): wrap each of the 15
   returned literals in `String(localized:)`.

Do NOT touch `MoreRoute.systemImage`, `statusKeyPath`, `MoreStatusKind.title`, or any other
property. `MoreStatusKind.title` (`rawValue.capitalized`) is a dynamic status string handled in
Wave 4 — leave it alone here.
  </action>
  <verify>
    <automated>test $(grep -c 'String(localized:' GooseSwift/AppShellView.swift) -ge 4 && test $(grep -c 'String(localized:' GooseSwift/MoreRouteModels.swift) -ge 30 && echo OK</automated>
  </verify>
  <done>GooseAppTab.title returns 4 String(localized:) values; MoreRoute.title and subtitle each return 15 String(localized:) values; no return types changed; no other properties modified.</done>
</task>
