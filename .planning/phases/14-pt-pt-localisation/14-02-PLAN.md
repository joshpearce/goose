Health metric families (Recovery V2, Sleep V2, Cardio, Strain, Stress, supplemental), and the
Coach view — into pt-PT by adding entries to `Localizable.xcstrings`.

screens need NO Swift code changes — only catalog entries whose key matches the existing English
literal. This wave covers ~150 of the static strings (D-01 comprehensive scope, D-05 Wave 2).

Purpose: These are the highest-traffic screens; localising them delivers the bulk of visible
pt-PT coverage.
Output: Expanded `Localizable.xcstrings` covering Home + Health + Coach static text.

  <action>
Enumerate the user-visible static string literals in these files and add a catalog entry for
each (key = exact English literal, `pt-PT` stringUnit with `state: translated`):
- GooseSwift/HomeDashboardView.swift
- GooseSwift/HealthDashboardViews.swift
- GooseSwift/HealthRecoveryStressViews.swift
- GooseSwift/HealthCardioViews.swift
- GooseSwift/HealthMetricFamilyStrainViews.swift

- GooseSwift/HealthSleepSheetsViews.swift
- GooseSwift/SleepV2BevelTrendViews.swift
- GooseSwift/SleepV2ScheduleViews.swift (static labels only)

Discovery command to inventory candidates:
`grep -rnoE '(Text|Section|Button|Label|LabeledContent|navigationTitle|accessibilityLabel)\("[^"]+"' GooseSwift/Home*.swift GooseSwift/Health*.swift GooseSwift/SleepV2*.swift`

Rules (D-01 / D-03 exclusions):
- INCLUDE: Text, Section, Button, Label, LabeledContent, navigationTitle, accessibilityLabel literals.
- EXCLUDE: `Text(verbatim:)` content (Pitfall 5), `record(source:title:body:)` log strings,
  `print()`/`Logger` strings, raw state-machine constants, SF Symbol names (systemImage:), and
  any string that is a value of a `@Published` status property (those are Wave 4 / D-04).
- EXCLUDE HealthDataStore+CoachSummaries.swift entirely — those feed the AI coach prompt context,
  not SwiftUI views (CONTEXT excludes them; A3 / Open Question 2).
- For interpolated strings, use the `%@` / `%lld` positional placeholder form as the key, NOT the
  Swift `\(...)` form (Pitfall 3). Use `%1$@`, `%2$@` ordering when multiple args appear.
- Reuse the glossary from Wave 1 (Recovery → Recuperação, Sleep → Sono, Save → Guardar,
  Cancel → Cancelar, Done → Concluído, Loading → A carregar..., Error → Erro, etc.). Do not create
  a duplicate key if Wave 1 already added it — keys are de-duplicated across the catalog.
  </action>
