**CRITICAL (N-02 — HKSampleQuery multi-fire completion handler):**

HKSampleQuery fires its completion handler MULTIPLE times: once per batch of results, and a final terminal call where `results == nil`. Using `withCheckedContinuation` and resuming on the first batch causes a fatal double-resume trap on the second call. Use this exact pattern for each query:

```
// Inside withCheckedContinuation body:
var accumulated: [HKQuantitySample] = []
let query = HKSampleQuery(sampleType: sampleType, predicate: nil, limit: HKObjectQueryNoLimit, sortDescriptors: [sortDescriptor]) { _, results, error in
    if let batch = results as? [HKQuantitySample] {
        accumulated.append(contentsOf: batch)
    }
    if results == nil {
        // Terminal call: results == nil means HKSampleQuery is done.
        // This is the ONLY place continuation.resume() is called.
        continuation.resume(returning: accumulated)
    }
}
store.execute(query)
```

Do NOT call `continuation.resume(...)` anywhere outside the `results == nil` branch. A guard on `results == nil` (not on `error`) is the correct termination signal for HKSampleQuery.

**CRITICAL (N-01 — Swift 6 actor isolation):**

HKSampleQuery completion handlers fire on an arbitrary background thread. The continuation body is therefore off-actor. The correct pattern:
1. Inside the continuation body (off-actor): accumulate `HKQuantitySample` objects into the local `var accumulated` array (local vars are fine off-actor).
2. Resume the continuation with the collected array (a value type — safe to cross actor boundary).
3. The `await withCheckedContinuation { ... }` expression returns on the `@MainActor` calling context — the entire function is already `@MainActor` so NO explicit `await MainActor.run { }` wrapper is needed.
4. After the continuation returns on-actor: iterate the accumulated array, extract tuples, and call `await upsertBodyComposition(...)` in a loop.

Do NOT call `await upsertBodyComposition(...)` inside the continuation body. Do NOT use `Task { @MainActor in ... }` inside the callback — the continuation resumption already returns to the `@MainActor` caller.

- bodyMass query (using the multi-fire pattern above): NSSortDescriptor(key: HKSampleSortIdentifierStartDate, ascending: false). After the continuation returns the accumulated array on-actor, for each HKQuantitySample: date = hkDateFormatter.string(from: sample.startDate), weightKg = sample.quantity.doubleValue(for: HKUnit.gramUnit(with: .kilo)). Call `await upsertBodyComposition(date:weightKg:bodyFatPct:muscleMassKg:source:)` for each.
- bodyFatPercentage query: same multi-fire pattern. Extract value = sample.quantity.doubleValue(for: HKUnit.percent()). Guard: `guard value >= 0.0 && value <= 1.0 else { continue }` (reject malformed samples). Multiply by 100.0 before passing as bodyFatPct (HealthKit returns 0–1 fraction; bridge expects 0–100 percent).
- Wrap the entire query + upsert sequence in `do { ... }`. On catch: set `self.importState = .failed(error.localizedDescription)` and return.
- After both upsert loops complete successfully: call `await loadBodyCompositionHistory()` once to refresh, then set `self.importState = .idle`.

`valueDomain` computed var: `(min: Double, max: Double)`. lo = points.min()!, hi = points.max()!. padding = max((hi - lo) * 0.3, 0.5). Return (min: lo - padding, max: hi + padding).

`chartPoint(index: Int, value: Double, plot: CGRect, domain: (min: Double, max: Double)) -> CGPoint` — CRITICAL (F-05 — y-inversion):
CoreGraphics coordinate system has y=0 at the TOP. Without inversion, higher weight values appear at the bottom and lower values appear at the top — the sparkline renders upside-down. Use the explicit inversion formula:
```
let normalized = (value - domain.min) / max(domain.max - domain.min, 0.001)
let x = plot.minX + CGFloat(index) / CGFloat(max(points.count - 1, 1)) * plot.width
let y = plot.maxY - CGFloat(normalized) * plot.height   // y=0 is TOP in CoreGraphics
return CGPoint(x: x, y: y)
```

`trendLine(in plot: CGRect) -> Path`: build Path by moving to chartPoint(0) then adding lines through all subsequent points. Uses `valueDomain`.

`envelopePath(in plot: CGRect) -> Path`: build closed Path with upper curve offset and lower curve offset. Spread per point = max((domain.max - domain.min) * 0.08, 0.2) kg. Upper path goes left-to-right with value + spread; lower path goes right-to-left with value - spread; close the path. Fill with tint.opacity(0.15).
