# Phase 51 Plan 01: Bug Audit — Fix HIGH & Critical MEDIUM Findings

## Goal
Corrigir todos os findings HIGH e os MEDIUM com risco de crash/corrupção de dados encontrados no code review das fases 36–50.

## Findings Summary

### HIGH (obrigatório corrigir)
| ID | File | Issue |
|----|------|-------|
| H-01 | GooseRustBridge.swift | Data race em `counter`/`lastTiming` sem lock quando múltiplos `requestValueAsync` correm em paralelo na mesma instância |

| H-03 | GooseUploadService.swift | `refreshPendingRowCount()` é público → pode ser chamado do @MainActor bloqueando main thread |

## Plan

### Step 1: Fix H-01 — GooseRustBridge data race
Add `NSLock` to guard `counter` and `lastTiming` mutations in `requestValue`.

### Step 2: Fix H-02 — importHistoricalDataFromServer shared rust instance
Replace `let bridge = rust` (shared) with `let bridge = GooseRustBridge()` (local instance).

### Step 3: Fix H-03 — refreshPendingRowCount public
Change `func refreshPendingRowCount()` to `private func refreshPendingRowCount()`.

### Step 4: Fix M-01 — NaN/Inf guard in V24Biometrics
Add `.isFinite` guard before `UInt16(clamping: Int(d))`.

### Step 5: Fix M-02 — Pagination loop guard
Add `maxPages = 200` guard to `importHistoricalDataFromServer` repeat-while loop.

### Step 6: Fix M-03 — Task @MainActor explicit
Change `Task { await self.syncBandSleepHistory() }` to `Task { @MainActor in await self.syncBandSleepHistory() }`.

### Step 7: Fix M-04+M-05 — exercise_detection.rs hardcoded strings + age fallback
- Replace `"fallback"` string comparison with a local constant
- Add quality flag when using age=30 proxy for hrmax

### Step 8: Fix M-06 — RFC3339 days-per-month validation
Add days-per-month lookup table to `parse_rfc3339_utc_unix_ms`.

### Step 9: Fix M-07 — Upload pending count decrement on Task.detached guard
Add decrement in guard let self else branch.
