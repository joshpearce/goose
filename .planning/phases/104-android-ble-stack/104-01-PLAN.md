# Plan 104-01: Android BLE GATT Client + Packet Framing + Rust Bridge Integration

## Objective

Implement `WhoopBleClient` — a Kotlin class that connects to a WHOOP device via
`BluetoothGatt`, subscribes to BLE notifications, reassembles Gen4 multi-notification
frames (same prepend-buffer algorithm as iOS `CoreBluetoothBLETransport+HistoricalHandlers`),
and forwards decoded frame hex strings to `GooseBridge.handle()` via `capture.import_frame_batch`.
Full iOS parity for connection state machine (6 states), auth sequence, and auto-reconnect.

## Context

**iOS parity target:** `GooseSwift/CoreBluetoothBLETransport.swift` + extensions
**Gen4 reassembly:** `GooseSwift/CoreBluetoothBLETransport+HistoricalHandlers.swift` (SYNC-09 prepend pattern)
**Android JNI bridge:** `android/app/src/main/kotlin/com/goose/app/bridge/GooseBridge.kt`
**Bridge method:** `capture.import_frame_batch` with `CapturedFrameInput` struct
**DB path:** `context.filesDir.absolutePath + "/goose.sqlite"` (D-07)

## Threat Model

<threat_model>
- T-104-01: Gen4 buffer unbounded growth from malformed frames → cap buffer at 8192 bytes (same as iOS)
- T-104-02: Bridge call blocks BLE callback thread → always dispatch to Dispatchers.IO
- T-104-03: CCCD descriptor write race (concurrent writes corrupt BLE state) → queue notifications, write one CCCD at a time
- T-104-04: Auth retry loop exhausts BLE connection → mirror iOS authRetryCount limit (12 cycles)
- T-104-05: Missing BLUETOOTH_SCAN/CONNECT permissions crash on API 31+ → declare in manifest with neverForLocation flag
</threat_model>

    Create `android/app/src/main/kotlin/com/goose/app/ble/FrameReassembler.kt`:

    ```kotlin
    class FrameReassembler(private val bufferCapBytes: Int = 8192) {
        private var buffer: ByteArray = ByteArray(0)

        // Feed new notification bytes; returns list of complete frames.
        // Gen4 uses this (multi-notification reassembly).
        // Gen5 callers pass bytes directly without using this class.
        fun feed(incoming: ByteArray): List<ByteArray>

        // Reset buffer (call on disconnect/reconnect).
        fun reset()
    }
    ```

    Implementation of `feed()` must exactly mirror the iOS SYNC-09 algorithm:
    1. Prepend buffer to incoming: `val input = buffer + incoming`
    2. Reset buffer to empty
    3. Call `parseFrames(input)` — walks bytes extracting complete frames:
       - Frame header: bytes [0]=type, [1]=len_lo, [2]=len_hi, [3]=seq
       - Frame total size = 4 + (len_lo | len_hi << 8)
       - If remaining bytes < declared total size → frame is incomplete; stop
    4. Compute `consumedCount` = sum of all extracted frame sizes
    5. Tail = input bytes after consumedCount; if tail.size > 0 && tail.size <= bufferCapBytes → store as new buffer; else clear buffer
    6. Return list of complete frame ByteArrays

    Create `android/app/src/test/kotlin/com/goose/app/ble/FrameReassemblerTest.kt` with:
    - Test: single complete frame in one notification → returns 1 frame
    - Test: frame split across two notifications → first notification returns 0 frames; second returns 1 complete frame
    - Test: two complete frames in one notification → returns 2 frames
    - Test: oversized tail (> 8192 bytes) is discarded, not stored in buffer
    - Test: `reset()` clears partial buffer; next notification treated as fresh start
    - Test: frame with declared length = 0 (empty body) is valid; 4-byte header returned

    Run tests: `./gradlew testDebugUnitTest` from `android/` directory.
  </action>

    GattCallback implementation:
    1. `onConnectionStateChange(STATE_CONNECTED)` → request MTU 247, update state to `Connecting`
    2. `onMtuChanged` → `gatt.discoverServices()`, update state to `DiscoveringServices`
    3. `onServicesDiscovered` → find WHOOP service (Gen4 or Gen5), queue CCCD enables for all notify chars, update state to `Authenticating`, call `sendAuthCommand()`
    4. `onDescriptorWrite` → pop next from `pendingCccdQueue`, write if any remain
    5. `onCharacteristicChanged` → if Gen4: reassemble via `gen4Reassembler.feed(value)` for each complete frame call `importFrame(frame)`; if Gen5: call `importFrame(value)` directly
    6. `onConnectionStateChange(STATE_DISCONNECTED)` → `gen4Reassembler.reset()`, update state to `Disconnected`, if `!userDisconnected` schedule reconnect after 5s cooldown via `scope.launch { delay(5000); reconnect() }`

    `sendAuthCommand()`: write WHOOP client hello bytes to command characteristic.
    The client hello is a fixed byte sequence that initiates the WHOOP auth handshake.
    Use the same bytes as the iOS implementation. Read `GooseSwift/CoreBluetoothBLETransport+Commands.swift`
    to extract the exact `clientHelloCommand` byte array and replicate it verbatim.
    If the auth characteristic write fails, increment `authRetryCount`; if `authRetryCount >= AUTH_RETRY_LIMIT` → set `authExhausted=true`, disconnect without reconnect.

    Auth success detection: the first notification received after `sendAuthCommand()` with a
    recognized packet type (non-error response from WHOOP) means auth succeeded → update state to `Connected`.

    `importFrame(frameBytes: ByteArray)`:
    ```kotlin
    private fun importFrame(frameBytes: ByteArray) {
        scope.launch(Dispatchers.IO) {
            val frameHex = frameBytes.joinToString("") { "%02x".format(it) }
            val dbPath = context.filesDir.absolutePath + "/goose.sqlite"
            val evidenceId = java.util.UUID.randomUUID().toString()
            val capturedAt = java.time.Instant.now().toString()
            val deviceModel = when(activeGeneration) {
                WhoopGeneration.GEN4 -> "whoop4"
                WhoopGeneration.GEN5 -> "whoop5"
                WhoopGeneration.MG -> "whoop_mg"
                null -> "unknown"
            }
            val request = buildImportRequest(dbPath, evidenceId, capturedAt, deviceModel, frameHex)
            GooseBridge.safeHandle(request)
        }
    }
    ```

    `buildImportRequest()`: constructs the JSON-RPC string:
    ```json
    {
      "schema": "goose.bridge.request.v1",
      "method": "capture.import_frame_batch",
      "args": {
        "database_path": "<dbPath>",
        "frames": [{
          "evidence_id": "<evidenceId>",
          "source": "android_ble",
          "captured_at": "<capturedAt>",
          "device_model": "<deviceModel>",
          "frame_hex": "<frameHex>",
          "sensitivity": "normal"
        }]
      }
    }
    ```
    Use `org.json.JSONObject` (available in Android SDK, no extra dep needed) or manual string building
    with proper escaping. Do NOT import a JSON library not already in `build.gradle.kts`.

    API 33 compatibility: `onCharacteristicChanged` has two signatures — implement both:

    - New override with `characteristic: BluetoothGattCharacteristic, value: ByteArray` for API 33+
    In the deprecated override, read `characteristic.value` to get bytes.

    MG detection: after connecting, if device name (from `BluetoothDevice.name`) contains " mg"
    (case-insensitive) AND service UUID is Gen5, set generation to `WhoopGeneration.MG`.
