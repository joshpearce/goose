<objective>
Implement the missing `POST /v1/ingest-frames` endpoint on the FastAPI server and make the raw-frame round-trip work end to end: frames uploaded via POST are retrievable via the existing `GET /v1/export/frames/{device_id}`.

The GET endpoint already exists (commit 6110dbf) but reads ONLY from `raw_batches` (file-based archive storage). Uploaded frames from the iOS app have no archive file, so we introduce a `raw_frames` TimescaleDB table to hold them, persist POSTed frames there, and extend `read_device_frames` to UNION results from both `raw_frames` and `raw_batches`. This closes ROUTE-01 (POST accepts + persists) and completes ROUTE-02 (uploaded frames are exportable, not just archive-derived frames).

Purpose: The iOS `GooseUploadService.uploadRawFrames` already POSTs to `/v1/ingest-frames` and reads `json["inserted"]`; today it gets 404/405. After this plan, the upload succeeds and `importHistoricalDataFromServer` can re-import the same frames on a fresh install.
Output: `raw_frames` schema, `insert_raw_frames_batch` store function, UNION in `read_device_frames`, the `POST /v1/ingest-frames` route, and a pytest module covering round-trip + auth + idempotency.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iOS client → POST /v1/ingest-frames | Untrusted JSON payload (frame_hex, device id) crosses into the server over a public tunnel |
| public internet → GET /v1/export/frames | Biometric raw frames exposed; must stay Bearer-gated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-46-01 | Tampering | `frame_hex` field on POST /v1/ingest-frames | mitigate | Pydantic `IngestFrame.frame_hex` validated with `pattern=r"^[0-9a-fA-F]*$"`; value persisted only via parametrised SQL (never string-formatted into the query) |
| T-46-02 | Information Disclosure | GET /v1/export/frames + POST /v1/ingest-frames | mitigate | Both routes carry `dependencies=[Depends(require_auth)]`; `secrets.compare_digest` Bearer check (existing middleware) — auth test asserts 401 |
| T-46-03 | Denial of Service | large frames array on POST | accept | Personal single-user server behind Bearer auth; iOS client caps upload at 2000 frames (GooseUploadService limit:2000). No additional cap added |
| T-46-04 | Tampering | SQL injection via device_id / column values | mitigate | All writes/reads use psycopg parametrised placeholders (`%s`); table/column names are hardcoded literals, never interpolated from request data |
</threat_model>
