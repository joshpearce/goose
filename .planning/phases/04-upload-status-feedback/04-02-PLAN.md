# Plan 04-02: Add "Status" Subsection to Remote Server Detail Screen

## Objective

Extend the `MoreRemoteServerViews.swift` detail screen (created in Phase 2) with a conditional "Status" subsection below the URL/token/toggle fields. The subsection is only visible when `uploadEnabled == true` and the server URL is configured. It displays three rows: server reachability (with system color), last upload timestamp (relative format via `RelativeDateTimeFormatter`), and pending batch count.

## Context

`MoreRemoteServerViews.swift` was created in Phase 2 as the detail screen for the "Remote Server" entry in the More tab. Phase 4 extends it to observe the three `@Published` properties added to `GooseAppModel` in plan 04-01. The screen already receives `GooseAppModel` as `@EnvironmentObject`.

<threat_model>
## Threat Model (ASVS L1)

| Threat | Mitigation |
|--------|-----------|
| Displaying stale server reachability as authoritative | Label copy clarifies "at launch" — no misleading current-state claim |
| Server URL visible in UI before being saved | Not displayed; only connectivity status is shown |
</threat_model>

Add a computed property `private var uploadIsActive: Bool` that returns `model.uploadEnabled && !model.serverURL.isEmpty` (using whatever UserDefaults-backed property names Phase 2 established for `uploadEnabled` and server URL).

Add a `Section("Status")` block below the existing configuration section, conditionally shown using `if uploadIsActive`:

```
Section("Status") {
  // Row 1: Server reachability (D-01, D-02)
  Label {
    switch model.serverReachable {
    case .none:
      Text("A verificar...").foregroundStyle(.secondary)
    case .some(true):
      Text("Servidor acessível").foregroundStyle(.green)
    case .some(false):
      Text("Servidor inacessível").foregroundStyle(.red)
    }
  } icon: {
    switch model.serverReachable {
    case .none:
      ProgressView().scaleEffect(0.7)
    case .some(true):
      Image(systemName: "checkmark.circle.fill").foregroundStyle(.green)
    case .some(false):
      Image(systemName: "xmark.circle.fill").foregroundStyle(.red)
    }
  }

  // Row 2: Last upload timestamp (D-05)
  if let lastUpload = model.lastUploadAt {
    LabeledContent("Último upload") {
      Text(RelativeDateTimeFormatter().localizedString(for: lastUpload, relativeTo: Date()))
        .foregroundStyle(.secondary)
    }
  } else {
    LabeledContent("Último upload") {
      Text("Nunca").foregroundStyle(.secondary)
    }
  }

  // Row 3: Pending batch count (D-06)
  LabeledContent("Batches pendentes") {
    Text("\(model.pendingBatchCount)")
      .foregroundStyle(model.pendingBatchCount > 0 ? .orange : .secondary)
  }
}
```

The `Section("Status")` is wrapped in `if uploadIsActive { ... }` so it is hidden when upload is disabled or server is not configured (D-07, specifics in CONTEXT.md).

Do NOT add `RelativeDateTimeFormatter` as a stored property — instantiate inline for simplicity (acceptable for a display-only label with infrequent updates).
