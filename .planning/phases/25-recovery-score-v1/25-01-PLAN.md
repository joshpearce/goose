<objective>
Implement the personal-baseline Recovery score in the Rust core: `goose_recovery_v1`
computes a weighted composite Z-score from the per-device EWMA baseline (HRV RMSSD +
resting HR), applies a logistic squash to a 0–100 score, and exposes trust level and
colour band. The function is reached from Swift through a new store-backed bridge method
`metrics.goose_recovery_v1`.

Purpose: ALG-REC-01 (Z-score + logistic squash, cold-start None) and ALG-REC-02 (trust
levels + colour bands) — the algorithmic foundation the Phase 25 dashboard consumes.
Output: New types and function in metrics.rs; new bridge dispatch arm + handler in bridge.rs;
unit tests for Z=0→58%, cold-start None, colour band boundaries, and trust level mapping.
</objective>
